# YouFlex 백엔드 구현 현황 (2026-07-18 기준)

`docs/project-plan.md`의 기능 정의서 순서를 기준으로, 실제 코드(컨트롤러/서비스/매퍼 인터페이스/매퍼 XML)를
하나씩 대조해서 조사한 결과입니다. "매퍼 인터페이스에는 메서드가 선언돼 있는데 XML에 구현이 없는" 경우가
꽤 있어서, 단순히 파일이 존재하는지가 아니라 **실제로 호출 가능한지**를 기준으로 판정했습니다.

상태 표기: ✅ 완료 · 🟡 부분 구현(일부만 되거나 껍데기만 있음) · ❌ 미구현(DTO만 있거나 아예 없음)

## 3-1. 회원 관리

- **회원가입** ✅ — `MemberController.join`, 아이디 중복확인 AJAX(`/join/check-id`) 정상.
- **로그인/로그아웃** ✅(일반) / ❌(소셜) — `login.js`의 카카오/구글 버튼은 `alert(provider + ' 계정으로 로그인되었습니다. (데모)')`만 호출, 실제 OAuth 연동 없음.
- **마이페이지**
  - 내 정보 ✅
  - 내 글 목록 ✅ (오늘 페이징 5개씩 구현: `MemberController.myReviews`, `ReviewMapper.findByMemberId`)
  - 북마크 ❌ — `BookmarkMapper.java` 인터페이스가 비어있고 XML 파일 자체가 없음. 컨트롤러도 없음.
  - 등업신청 🟡 — 버튼 누르면 신청 상태로 바뀌고 관리자가 승인/반려 가능(`AdminMemberController`)하지만, 기획서 조건(게시글 3회·경고 0회·가입 7일 경과)은 서버에서 검증하지 않음.
  - 취향 변경 ✅
- **회원정보 수정** ✅ — 오늘 발견한 병합 충돌 유실(`updateProfile` 호출 자체가 사라졌던 버그) 복구 완료.
- **회원탈퇴신청** ✅ — 오늘 구현(`member_delete_status`를 '탈퇴'로 전환, 탈퇴 계정 로그인 차단 포함).

## 3-2. 리뷰 게시판 — 가장 구현이 덜 된 영역

- **작성** ✅ — `ReviewController.write` → `review` INSERT + `review_mapping`(다대다 장르) INSERT.
- **수정/삭제** ❌ — `ReviewMapper.update`/`delete` 메서드는 선언돼 있지만 XML 구현이 없고, 이걸 호출하는 컨트롤러도 없음.
- **임시저장** ❌ — `ReviewDraftMapper.xml`이 완전히 빈 파일.
- **검색(전체/상세)** ❌ — 어떤 컨트롤러에도 검색 엔드포인트가 없음.
- **목록 조회/상세페이지** ❌ — `ReviewMapper.findAll`/`countAll`/`findById` 전부 XML 구현 없음. `templates/review/`에는 `write.html`밖에 없고, 목록/상세 화면은 `html/04_list.html`, `html/13_detail.html` 목업 상태로만 존재(실제 서비스에 미연결). 조회수(`review_hit`) 증가 로직도 없음.
- **별점 평가** 🟡 — 작성 시 입력은 받지만, 상세페이지 자체가 없어서 "평가"라는 행위가 실제로 노출되는 화면이 없음.
- **좋아요/북마크** ❌ — `ReviewLikeMapper.xml` 빈 파일, `BookmarkMapper`는 인터페이스도 빈 상태.
- **베스트글** ❌ — 관련 코드 없음.
- **신고하기** ❌ — `ReviewReportMapper`에 `selectReviewReportList`/`updateReviewReportStatus`(관리자용 조회·처리)만 있고, "신고 등록"에 해당하는 insert 메서드 자체가 없음. 즉 신고를 접수할 방법이 없어서, `sql/test_data_reports.sql`로 직접 DB에 꽂아 넣어야만 관리자 신고 관리 화면에서 볼 수 있는 상태.
- **공지사항 게시판** ✅ — `NoticeController`/`NoticeViewController` CRUD 완비, 최상단 고정 여부는 프론트 확인 필요.
- **비속어 필터** ❌ — `BadWordDTO`만 있고 매퍼/서비스/필터링 로직 전혀 없음. 게시글·댓글·채팅 어디에도 적용 안 됨.
- **Q&A 게시판** ✅ — `QnaController`(글/댓글/신고) + `AdminAnswerController`(관리자 답변)까지 CRUD 완비. 이 프로젝트에서 가장 완성도 높은 게시판.

## 3-3. 댓글 시스템 — 사실상 미구현

- `CommentMapper`, `CommentLikeMapper` 인터페이스가 **메서드 하나도 선언 안 된 빈 껍데기**이고 XML도 빈 파일.
- `CommentReportMapper`만 유일하게 구현돼 있는데, 이것도 관리자 조회/처리용(`selectCommentReportList`/`updateCommentReportStatus`)일 뿐 댓글 작성/신고 접수 경로가 없음.
- 컨트롤러 자체가 아예 없음(`CommentController` 없음).
- → 댓글 작성/수정/삭제/대댓글/좋아요/베스트댓글 전부 미구현.

## 3-4. AI 챗봇(퀴즈)

- ❌ — `QuizDTO`/`QuizAttemptDTO`만 있고 매퍼/서비스/컨트롤러 전혀 없음. `app.js`의 퀴즈는 문제은행이 JS에 하드코딩(`QNA_QUIZ_QUESTION_BANK`)돼 있고, 하루 3회 제한도 서버가 아니라 `localStorage`로 처리 중이라 브라우저 저장소만 지우면 무제한 시도 가능. 정답 시 포인트 적립도 실제로 일어나지 않음(포인트 시스템 자체가 미구현).

## 3-5. 실시간 채팅방

- **개설/목록/삭제/참여** ✅ — `ChatroomController`, `ChatroomMapper`, `ChatMemberMapper` 다 구현됨.
- **실시간 메시지 송수신** 🟡 — `WebSocketChatController`가 STOMP로 브로드캐스트는 하지만, 메시지를 `chat_message` 테이블에 저장하는 부분은 주석(`// chatService.saveMessage(message);`)으로 비활성화돼 있어 새로고침하면 대화 내용이 사라짐.
- **최대인원제한/방장퇴장시 삭제/강퇴/경고표시/입퇴장 시스템 메시지** ❌ — 화면상 버튼(`giveChatWarning()` 등)은 있지만 전부 `alert`/`prompt` 기반 프론트 mock이고 `chat_warning` 테이블을 쓰는 코드가 없음.

## 3-6. 알림 시스템 / 관리자 페이지

- **알림** ❌ — `NotificationsDTO`만 있고 매퍼/서비스/컨트롤러 없음. 헤더의 🔔 배지도 정적 숫자.
- **회원 관리** ✅, **신고 처리(관리자 쪽)** ✅, **공지사항 관리** ✅, **등업신청 관리** ✅, **채팅방 강제 삭제** ✅, **경고 부여(3회 누적 자동 강제탈퇴 포함)** ✅ — `WarningService.issueWarning`에 정책 §5-4 그대로 구현돼 있음.
- **배너 관리** ❌ — `admin.html`에 화면은 있지만 `admin.js`의 `saveBanner()`가 `alert("배너가 저장되었습니다. (데모)")`만 하고 DOM만 조작, `BannerDTO` 외엔 백엔드 코드 없음.
- **가입/탈퇴 통계 대시보드** ❌ — `admin.html`에 "대시보드 요약" 섹션은 있지만 실제 통계를 집계하는 서버 코드가 없음(정적 화면으로 추정).
- **Q&A 답변** ✅.

## 3-7. 포인트 시스템

- ❌ (전체) — `PointHistoryDTO`만 있고 매퍼/서비스가 전혀 없음. 좋아요·퀴즈 정답으로 포인트가 적립되는 코드도, 경고차감/하이라이트 구매로 포인트가 차감되는 코드도 없음. 마이페이지 "포인트 상점" 탭 구매 버튼들도 `alert(...데모...)`만 함.

## 3-8. 예외 처리

- ❌ — 404/500 커스텀 에러 페이지 템플릿이 존재하지 않음(Spring Boot 기본 에러 페이지로 노출될 것으로 추정).

## 정책(§5) 구현 여부

- **§5-1 등급 정책**: 등급 자체(시청자/평론가/관리자)와 등급별 화면 노출 조건은 잘 반영돼 있음. 다만 평론가 승급 **자격 조건**(게시글 3회/경고 0회/가입 7일)은 서버에서 검증하지 않음.
- **§5-2 약관 동의**: 회원가입 화면에 체크박스는 있으나(`join.html`) 서버에서 필수 체크 여부를 검증하는지는 별도 확인 필요.
- **§5-3 포인트 정책**: 전체 미구현(위 3-7 참고).
- **§5-4 경고/제재**: 경고 부여 시 3회 누적 자동 강제탈퇴는 구현됨. 다만 "포인트로 경고 차감"은 관리자 수동 차감(`revokeWarning`)만 있고, 회원이 마이페이지 포인트 상점에서 직접 차감권을 구매하는 흐름은 데모 alert뿐이라 실제로는 연결 안 됨.

## 기타 참고사항

- 이번 세션 중 실제 운영에 영향 있는 버그 두 건을 발견해서 고쳤습니다: `member_grade`/취향 매핑 테이블명이 DB에서 리네임됐는데 코드가 못 따라간 것(회원가입 실패 원인), 병합 충돌 해결 과정에서 `updateMyPage`의 저장 로직이 통째로 날아갔던 것.
- `WebConfig`에 `/upload/**` 리소스 핸들러가 없어서, 게시글 작성 시 업로드한 이미지가 있어도 화면에 표시할 방법이 없습니다(파일 저장 자체는 됨).
- `html/` 폴더에 있는 정적 목업(`04_list.html`, `13_detail.html` 등)은 `static/js/detail.js` 등 대응하는 JS와 함께 아직 `templates/`로 이전되지 않은 상태 — 이 마이그레이션이 곧 "리뷰 게시판 목록/상세 화면 구현"과 직결됩니다.
